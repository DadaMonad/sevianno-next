Class: Sevianno

Defined in: lib\sevianno.coffee

Overview

This is the class that all sevianno tools use. After initialization a

  • iwc Client is initialized
  • las Client is initialized
  • log in / out It provides some convenient wrappers of functions.

# (void) registerLasFeedbackHandler(statusCode, f)

Register new las feedback handler which listens to $statusCode. It is possible to add multiple handlers for one statusCode. $f is executed with: f(statusCode, message)

Examples:

registerLasFeedbackHandler 'LoginSuccess', onLogin

# (void) registerIwcCallback(actionName, f)

Register new iwc feedback handler which listens to $actionName. It is possible to add multiple handlers for one action name. $f is executed with: f(statusCode, message)

Examples:

registerLasFeedbackHandler('LoginSuccess', function(statusCode, message){})

# (void) sendIwcIntent(intent)

Publish an iwc intent

Examples:

intent = {
  "component":"",
  "action":"ACTION_LOGIN",
  "data":"http://example.org",
  "dataType":"text/html",
  "categories":["example1","example2"],
  "flags":["PUBLISH_LOCAL"],
  "extras":{"sessionId":sessionId, "user":user}
}

# (void) getVideoInformation()

Get all the videoinformation in the database. A 'videoinformation' is a string encoded xml. Parse it e.g. with jquery. For more information about a 'videoinformation' consult the documentation of the las library.

See also:

  • http://dbis.rwth-aachen.de/~jahns/javadocs/videoinformation/
